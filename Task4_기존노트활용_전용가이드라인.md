## 개요

기존 노트의 발견, 분석, 품질 평가, 중복 처리, 통합 전략에 관한 체계적인 가이드라인입니다. 신규 지식 구조화 시 기존 자료를 최대한 활용하여 효율성을 높이고 지식 네트워크의 일관성을 유지하는 것이 목적입니다.

## 1. 기존 노트 발견 전략

### 1.1 다층적 검색 방법론

#### A. 제목 기반 검색 (1차 스크리닝)
```
검색 단계:
1. 핵심 키워드로 직접 검색
   - 주제명, 기술명, 개념명 등
   - 예: "Python", "머신러닝", "데이터베이스"

2. 관련 키워드로 확장 검색  
   - 동의어, 유사어, 영문/한글 변형
   - 예: "ML", "기계학습", "딥러닝" (머신러닝 관련)

3. 상위/하위 개념으로 확장
   - 상위: "프로그래밍", "인공지능"
   - 하위: "scikit-learn", "TensorFlow"
```

#### B. 태그 기반 탐색 (2차 정밀 검색)
```
태그 탐색 전략:
1. 직접 관련 태그 검색
   - #python, #javascript, #데이터분석

2. 분야별 태그 조사
   - #backend, #frontend, #algorithm
   - #전자기학, #회로이론 (전기 관련)

3. 상태/유형 태그 확인
   - #completed, #in-progress
   - #project, #reference, #code

AI 지원 프롬프트:
"[주제명] 관련 기존 노트를 찾기 위해 다음 태그들을 검색합니다:
직접 관련: #[주제-태그1], #[주제-태그2]
분야 관련: #[분야-태그1], #[분야-태그2]  
상태 확인: #completed, #in-progress
검색 결과를 정리해드릴까요?"
```

#### C. 내용 기반 유사도 검색 (3차 세밀 검색)
```
내용 검색 전략:
1. 핵심 개념어로 본문 검색
   - 정의, 특징, 원리 등의 내용
   - 예: "딥러닝은", "신경망의 특징"

2. 코드 블록 내 검색 (기술 노트)
   - 함수명, 라이브러리명, 클래스명
   - 예: "import tensorflow", "class MyModel"

3. 참고 자료 URL 또는 출처 검색
   - 동일 출처에서 나온 다른 내용들
   - 저자, 도서명, 웹사이트 등

검색 우선순위:
높음: 제목 정확 일치 > 태그 정확 일치
중간: 제목 부분 일치 > 내용 핵심 개념 일치  
낮음: 태그 관련 일치 > 참고 자료 일치
```

#### D. 링크 관계 역추적 (4차 연결망 검색)
```
링크 추적 방법:
1. 백링크 조사
   - 현재 검색된 노트를 참조하는 다른 노트들
   - Obsidian의 백링크 패널 활용

2. 출처 링크 추적
   - 같은 참고 자료를 인용한 다른 노트들
   - Reference Note들의 교차 참조

3. MOC 연결 추적
   - 관련 MOC에 연결된 다른 노트들
   - 계층적 MOC 구조 내 관련 노트

AI 지원 프롬프트:
"발견된 노트 [[노트명]]의 연결 관계를 분석합니다:
- 이 노트를 참조하는 노트: [백링크들]
- 이 노트가 참조하는 노트: [링크들]  
- 같은 MOC에 속한 노트: [MOC-연결-노트들]
추가로 조사할 관련 노트들을 정리해드릴까요?"
```

### 1.2 발견 효율성 체크리스트

```
✅ 검색 완료 기준:
□ 주제 핵심 키워드 3가지 이상 검색 완료
□ 관련 태그 5개 이상 조사 완료  
□ 발견된 노트의 백링크 확인 완료
□ 관련 MOC 2개 이상 조사 완료
□ 최근 3개월 내 작성 노트 우선 확인 완료

⚠️ 추가 검색 필요 신호:
□ 예상보다 발견된 노트가 적음 (3개 미만)
□ 발견된 노트들이 모두 오래됨 (6개월 이상)
□ 핵심 개념이 다뤄지지 않음
□ 관련 MOC가 발견되지 않음
```

## 2. 기존 노트 품질 평가 기준

### 2.1 4차원 품질 평가 매트릭스

#### A. 완성도 평가 (Completeness) - 25%
```
평가 항목별 점수 (각 5점 만점, 총 25점):

1. Frontmatter 완성도 (5점)
   - 5점: 모든 필수 필드 완성 + 선택 필드 일부
   - 4점: 모든 필수 필드 완성
   - 3점: 핵심 필드 (title, note-type, created) 완성
   - 2점: 일부 필수 필드 누락
   - 1점: frontmatter 없거나 심각한 오류

2. 내용 충실도 (5점)
   - 5점: 주제를 충분히 다루고 예시/설명 풍부
   - 4점: 주제의 핵심 내용 포함
   - 3점: 기본적인 내용은 있으나 부족
   - 2점: 내용이 매우 간략함
   - 1점: 제목만 있거나 내용 거의 없음

3. 구조화 수준 (5점)
   - 5점: 헤더 체계 완벽, 논리적 흐름 우수
   - 4점: 기본적인 헤더 구조 있음
   - 3점: 일부 구조화되어 있음
   - 2점: 구조화 시도는 있으나 일관성 부족
   - 1점: 구조화되지 않은 단순 나열

4. 링크 연결도 (5점)
   - 5점: 관련 노트와 풍부하게 연결됨
   - 4점: 주요 관련 노트와 연결됨
   - 3점: 기본적인 연결 있음
   - 2점: 최소한의 연결만 있음
   - 1점: 다른 노트와 연결 없음

5. 5분 규칙 준수 (5점)
   - 5점: 5분 내 완전히 이해 가능
   - 4점: 약간의 배경지식으로 이해 가능
   - 3점: 보통 수준의 이해 시간 필요
   - 2점: 상당한 집중이 필요함
   - 1점: 이해하기 매우 어려움
```

#### B. 정확성 평가 (Accuracy) - 25%
```
평가 항목별 점수 (각 5점 만점, 총 25점):

1. 정보 신뢰성 (10점)
   - 9-10점: 출처 명확, 정보 검증됨
   - 7-8점: 출처 있으나 일부 미검증
   - 5-6점: 출처 불명확하나 내용 합리적
   - 3-4점: 일부 의심스러운 정보
   - 1-2점: 정확성에 심각한 문제

2. 최신성 (10점)
   - 9-10점: 최신 정보 반영 (6개월 이내)
   - 7-8점: 비교적 최신 (1년 이내)
   - 5-6점: 보통 수준 (2년 이내)
   - 3-4점: 다소 오래됨 (3년 이내)
   - 1-2점: 매우 오래되거나 구식 정보

3. 출처 명시 (5점)
   - 5점: 모든 정보에 출처 명시
   - 4점: 주요 정보에 출처 있음
   - 3점: 일부 출처 명시
   - 2점: 출처 거의 없음
   - 1점: 출처 전혀 없음
```

#### C. 활용 가능성 평가 (Usability) - 25%
```
평가 항목별 점수 (각 12.5점 만점, 총 25점):

1. 재사용성 (12.5점)
   - 11-12.5점: 다른 프로젝트에서 직접 활용 가능
   - 9-10점: 약간의 수정으로 재사용 가능
   - 6-8점: 참고 자료로 활용 가능
   - 3-5점: 아이디어 수준에서만 도움
   - 1-2점: 재사용 가치 낮음

2. 확장 가능성 (12.5점)
   - 11-12.5점: 새로운 내용 추가 매우 용이
   - 9-10점: 확장 구조 갖춤
   - 6-8점: 일부 확장 가능
   - 3-5점: 확장 어려우나 가능
   - 1-2점: 확장 매우 어려움
```

#### D. 통합 적합성 평가 (Integration) - 25%
```
평가 항목별 점수 (각 12.5점 만점, 총 25점):

1. 스타일 일관성 (12.5점)
   - 11-12.5점: 현재 스타일 가이드와 완전 일치
   - 9-10점: 대부분 일치, 소폭 수정 필요
   - 6-8점: 일부 일치, 부분 수정 필요
   - 3-5점: 상당한 스타일 수정 필요
   - 1-2점: 전면적인 스타일 개편 필요

2. 주제 적합성 (12.5점)
   - 11-12.5점: 새 주제와 직접적 관련성
   - 9-10점: 강한 관련성
   - 6-8점: 보통 수준의 관련성
   - 3-5점: 약한 관련성
   - 1-2점: 관련성 거의 없음
```

### 2.2 종합 품질 등급 분류

```
S급 (90-100점): 우수 - 즉시 활용 가능
- 대부분의 영역에서 고득점
- 최소한의 수정으로 통합 가능
- 새로운 지식의 기반 자료로 활용

A급 (80-89점): 양호 - 경미한 개선 후 활용
- 일부 영역에서 개선 필요
- 내용은 우수하나 형식 정리 필요
- 주 참고 자료로 활용 가능

B급 (70-79점): 보통 - 부분적 개선 후 활용  
- 여러 영역에서 보완 필요
- 내용 중 일부만 활용 가능
- 보조 참고 자료로 활용

C급 (60-69점): 미흡 - 대폭 개선 필요
- 대부분 영역에서 개선 필요
- 아이디어나 구조만 참고
- 전면 재작성 고려

D급 (60점 미만): 부적합 - 활용 곤란
- 품질이 현저히 낮음
- 새로운 작업에 도움 안 됨
- 아카이브 또는 삭제 고려
```

## 3. 중복 노트 식별 및 처리 전략

### 3.1 중복도 측정 기준

#### A. 제목 유사도 (30%)
```
유사도 계산:
- 완전 일치: 100%
- 핵심 키워드 80% 이상 일치: 90%
- 핵심 키워드 60% 이상 일치: 70%
- 핵심 키워드 40% 이상 일치: 50%
- 핵심 키워드 20% 이상 일치: 30%

예시:
- "Python 기초 문법" vs "파이썬 기본 문법": 90%
- "머신러닝 개념" vs "기계학습 기초": 70%  
- "데이터베이스 설계" vs "DB 구조": 50%
```

#### B. 내용 중복도 (50%)
```
중복 내용 비율:
- 80% 이상 동일: 중복 심각 (통합 필수)
- 60-79% 동일: 중복 높음 (통합 권장)  
- 40-59% 동일: 중복 보통 (선택적 통합)
- 20-39% 동일: 중복 낮음 (연결만 강화)
- 20% 미만: 중복 없음 (별도 관리)

내용 분석 요소:
- 핵심 개념 일치도
- 예시/설명 중복도  
- 코드 블록 유사도
- 참고 자료 중복도
```

#### C. 태그 및 분류 유사도 (20%)
```
분류 유사도:
- 동일 note-type + 주요 태그 일치: 90%
- 동일 note-type + 일부 태그 일치: 70%
- 다른 note-type + 주요 태그 일치: 50%
- 다른 note-type + 일부 태그 일치: 30%
- 완전히 다른 분류: 0%
```

### 3.2 중복 처리 의사결정 매트릭스

#### A. 통합 필요 (총 중복도 70% 이상)
```
처리 방식 결정:
1. 품질 차이가 큰 경우
   - 고품질 노트를 기반으로 저품질 노트 내용 흡수
   - 저품질 노트는 아카이브 또는 삭제

2. 품질이 비슷한 경우  
   - 더 완성도 높은 노트에 다른 노트 내용 병합
   - 중복 부분 제거, 고유 부분만 추가

3. 각각 고유 가치가 있는 경우
   - 상위 개념으로 통합 노트 신규 생성
   - 기존 노트들은 세부 항목으로 연결
```

#### B. 연결 강화 (중복도 40-69%)
```
처리 방식:
1. 상호 참조 링크 추가
   - 각 노트에서 관련 노트로 링크
   - "관련 노트" 섹션에 명시적 연결

2. 상위 MOC에서 관계 명시
   - MOC에서 두 노트의 관계 설명
   - 어떤 차이점과 연결점이 있는지 명시

3. 태그 통일
   - 공통 주제에 대해 일관된 태그 사용
   - 세부 차이점은 별도 태그로 구분
```

#### C. 별도 관리 (중복도 40% 미만)
```
처리 방식:
1. 현재 상태 유지
   - 각자의 고유성 인정
   - 필요시에만 상호 참조

2. 분류 체계 정리
   - 서로 다른 관점임을 태그나 제목에 명시
   - 예: "Python 문법", "Python 실습"
```

### 3.3 중복 처리 AI 지원 프롬프트

```
AI 중복 분석 프롬프트:

## 4. 기존 MOC와의 통합 전략

### 4.1 MOC 관계 유형 분석

#### A. 확장형 통합 (가장 일반적)
```
상황: 기존 MOC에 새로운 하위 주제 추가
예시: "Python 기초" MOC → "Python 고급" 내용 추가

처리 방법:
1. 기존 MOC 구조 분석
   - 현재 하위 주제들 파악
   - 논리적 흐름 이해
   - 확장 가능한 지점 식별

2. 새 내용의 적절한 위치 결정
   - 기존 구조와의 논리적 연결점
   - 난이도나 학습 순서 고려
   - 기존 섹션 vs 새 섹션 판단

3. 점진적 통합 실행
   - 기존 링크들의 유효성 확인
   - 새 노트들과 기존 노트들 연결
   - MOC 전체 흐름 재검토
```

#### B. 병합형 통합
```
상황: 여러 개별 노트들을 하나의 체계로 통합
예시: "AI 개념", "ML 기초", "딥러닝 입문" → "인공지능 통합" MOC

처리 방법:
1. 기존 노트들의 관계 매핑
   - 상하위 관계 파악
   - 중복 영역 식별
   - 빈 영역(gap) 발견

2. 새로운 통합 구조 설계
   - 논리적 계층 구조 수립
   - 각 기존 노트의 새로운 위치 결정
   - 추가 필요한 브릿지 노트 계획

3. 단계적 통합 실행
   - 상위 MOC 먼저 생성
   - 기존 노트들 순차적 연결
   - 새로운 연결고리 노트 생성
```

#### C. 분할형 재구조화
```
상황: 기존 MOC가 너무 복잡해져서 분할 필요
예시: "웹 개발" MOC → "프론트엔드", "백엔드" 별도 MOC

처리 방법:
1. 분할 기준 명확화
   - 주제별, 기술별, 난이도별 등
   - 각 분할 영역의 독립성 확보
   - 상호 연결 지점 파악

2. 새로운 MOC 구조 설계
   - 각 분할 MOC의 범위 정의
   - 상위 MOC와 하위 MOC 관계 설정
   - 교차 참조 영역 처리 방안

3. 점진적 마이그레이션
   - 기존 노트들의 새로운 MOC 배정
   - 링크 관계 업데이트
   - 기존 MOC의 역할 재정의
```

### 4.2 MOC 통합 체크리스트

#### 통합 전 준비사항
```
✅ 기존 MOC 분석 완료
□ MOC 내 노트 목록 정리
□ 각 노트의 품질 상태 파악
□ 현재 링크 구조 매핑
□ 빈 영역(gap) 식별

✅ 통합 계획 수립 완료  
□ 통합 방식 결정 (확장/병합/분할)
□ 새 구조 설계 완료
□ 단계별 작업 계획 수립
□ 예상 소요 시간 산정

✅ 백업 및 안전장치
□ 기존 MOC 백업 완료
□ 중요 노트들 별도 백업
□ 실수 시 복구 방안 준비
```

#### 통합 후 검증사항
```
✅ 구조적 완성도 검증
□ 모든 기존 노트가 적절히 배치됨
□ 새로운 논리적 흐름이 자연스러움
□ 중복이나 빈 영역이 해결됨
□ 3단계 이하 깊이 유지

✅ 연결성 검증
□ 모든 링크가 정상 작동
□ 상호 참조가 적절히 설정됨
□ MOC와 개별 노트 간 연결 완료
□ 백링크 관계 정상

✅ 품질 일관성 검증
□ frontmatter 규칙 통일 적용
□ 스타일 가이드 일관성 유지
□ 태그 체계 정리 완료
□ 5분 규칙 준수 확인
```

## 6. AI 자동화 지원 확장 방안

### 6.1 기존 노트 자동 발견 시스템

#### 스마트 검색 알고리즘
```
AI 지원 기능:
1. 의미 기반 검색
   - 키워드 동의어/유의어 자동 확장
   - 개념 간 관계 이해 기반 검색
   - 맥락 고려한 관련도 계산

2. 패턴 인식 검색
   - 사용자 기존 노트 패턴 학습
   - 명명 규칙, 태그 사용 패턴 분석
   - 개인화된 검색 전략 제안

3. 시간적 관련성 고려
   - 최근 학습 이력 반영
   - 계절성이나 프로젝트 주기 고려
   - 관심 영역 변화 패턴 학습
```

### 6.2 품질 평가 자동화

#### 자동 품질 스코어링
```
AI 평가 시스템:
1. 구조적 분석
   - frontmatter 완성도 자동 체크
   - 헤더 구조 논리성 평가
   - 링크 밀도 및 적절성 측정

2. 내용 분석  
   - 텍스트 복잡도 측정 (5분 규칙)
   - 정보 밀도 계산
   - 예시/설명 비율 분석

3. 일관성 검증
   - 스타일 가이드 준수도 체크
   - 기존 노트들과의 일관성 비교
   - 태그 사용 패턴 일관성 확인
```

### 6.3 통합 제안 시스템

#### 스마트 통합 추천
```
AI 추천 시스템:
1. 구조 최적화 제안
   - MOC 구조 개선 방안 제시
   - 링크 추가/제거 제안
   - 노트 병합/분할 권고

   - 빈 영역(gap) 자동 식별
   - 관련 외부 자료 추천
   - 예시나 설명 보완 제안

3. 워크플로우 최적화
   - 사용자 패턴 기반 맞춤 제안
   - 효율적 작업 순서 추천
   - 시간 소모 예측 및 최적화
